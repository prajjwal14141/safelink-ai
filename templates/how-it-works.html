Behind the AI: How We Scan URLs

SafeLink AI employs a sophisticated Machine Learning pipeline, trained on a massive dataset of over one million URLs, to distinguish safe links from malicious ones. This process involves several key steps:

1. URL Normalization

Before analysis, every submitted URL undergoes a standardization process. This "cleaning" removes parts that are generally uninformative for risk assessment, ensuring the AI focuses on the core components:

  • Protocols like http:// and https:// are removed.
  • Common prefixes such as www. are stripped away.
  • Any trailing slashes (/) at the end of the URL are eliminated.
  • Example: The URL https://www.dangerous-site.com/login/ is normalized to dangerous-site.com/login.

2. Feature Engineering

The normalized URL is then dissected to extract meaningful "features" – the specific characteristics the AI model uses to make its judgment:

Lexical Tokens

The URL is segmented into constituent "words" (tokens) using delimiters like /, -, and .. Common, low-value tokens (com, org) are filtered out. The AI learns which tokens are statistically associated with malicious versus benign URLs.

Example: free-prize.tk/login.php → ['free', 'prize', 'tk', 'login', 'php']

Shannon Entropy

We calculate the URL's entropy, quantifying its character-level randomness. Malicious URLs often exhibit high entropy due to auto-generated strings designed to evade simple blacklist filters.

Higher score = more random (e.g., x3zr9.xyz). Lower score = simpler (e.g., google.com).

3. Machine Learning Classification

The core of SafeLink AI is a Logistic Regression model. This algorithm takes the extracted features (token presence/absence, entropy score) as input. During training, it learned the optimal "weights" or importance for each feature based on the large dataset.

When analyzing a new URL, the model combines the weighted features to compute a probability score indicating the likelihood of the URL being malicious. Based on a predefined threshold, it outputs the final classification: Safe or Malicious.

Importantly, the model considers all features together. A single suspicious token or high entropy might not be enough to flag a URL if other features strongly indicate it's safe. It's the combined pattern that matters.
